## What This Epic Is

Tracks full remediation of all findings from two Invicti Standard security scans:
- **DEV scan:** March 17, 2026 — `caninecommons-dev.cancer.gov` — 1,051,327 requests
- **Stage scan:** February 3, 2026 — `caninecommons-stage.cancer.gov` — 923,441 requests

Findings from both scans have been reconciled into a unified ticket set under this epic.

## Critical Tomcat Vulnerability — Pre-Sprint Hotfix (NOT in ICDC-4120)

A separate Invicti scan of `caninecommons-stage.cancer.gov` (February 3, 2026) identified a **Critical** severity finding: Apache Tomcat 10.1.31 affected by **CVE-2024-56337** (TOCTOU race condition — incomplete mitigation of CVE-2024-50379).

This was addressed as an **emergency hotfix** outside the Sprint 43 cadence:
- **Ticket:** ICDC-4014 (`CVE: Tomcat/jdk upgrade for backend microservice`)
- **Epic:** ICDC-3492 (separate epic — not ICDC-4120)
- **Assignee:** Eric Miller
- **Status:** Closed ✅
- **Fix:** Upgraded Tomcat to 10.1.34, deployed to all tiers (DEV, QA, Stage, Prod)
- **Link:** ICDC-4120 has a "Related To" link to ICDC-4014 for traceability

> When working ICDC-4120, do NOT create a new Tomcat upgrade ticket — ICDC-4014 already tracks this work and is closed.

## TLS Finding — Source Reconciliation

The DEV scan flagged **TLS 1.1** as Low severity. The Stage scan confirmed **TLS 1.0** as a **High/Confirmed** finding. ICDC-4127 has been upgraded to High priority to reflect this. TLS 1.0 is strictly worse — it enables the BEAST attack and has been non-compliant with PCI DSS since June 2018. The fix (enforce TLS 1.2+ minimum) eliminates both TLS 1.0 and TLS 1.1 simultaneously.
