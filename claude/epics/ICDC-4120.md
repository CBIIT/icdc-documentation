## What This Epic Is

Tracks full remediation of all findings from an Invicti Standard security scan of `caninecommons-dev.cancer.gov`, conducted on **March 17, 2026**. Invicti is an automated web application security scanner used across NCI/CBIIT. The scan issued 1,051,327 requests over 3 hours and 2 minutes.

**Total findings: 25 identified, 7 confirmed.**

## Critical Tomcat Vulnerability — Pre-Sprint Hotfix (NOT in ICDC-4120)

A separate Invicti scan of `caninecommons-stage.cancer.gov` (February 3, 2026) identified a **Critical** severity finding: Apache Tomcat 10.1.31 affected by **CVE-2024-56337** (TOCTOU race condition — incomplete mitigation of CVE-2024-50379).

This was addressed as an **emergency hotfix** outside the Sprint 43 cadence:
- **Ticket:** ICDC-4014 (`CVE: Tomcat/jdk upgrade for backend microservice`)
- **Epic:** ICDC-3492 (separate epic — not ICDC-4120)
- **Assignee:** Eric Miller
- **Status:** Closed ✅
- **Fix:** Upgraded Tomcat to 10.1.34, deployed to all tiers (DEV, QA, Stage, Prod)
- **Link:** ICDC-4120 has a "Related To" link to ICDC-4014 for traceability

> When working ICDC-4120, do NOT create a new Tomcat upgrade ticket — ICDC-4014 already tracks this work and is closed.

## Promotion Strategy

All remediations follow the standard ICDC SDL:

```
DEV → QA → Stage → Prod
```

Fixes are developed and validated in DEV first. No production changes occur until each fix is independently verified in QA and Stage.

## Key Technical Context

- **Frontend:** React web application
- **Backend:** Apache Tomcat 10.1.34 (upgraded via hotfix ICDC-4014) + Hasura GraphQL gateway
- **Infrastructure:** Nginx reverse proxy
- **Analytics:** Adobe DTM/Launch script loaded from `assets.adobedtm.com`
- **External resources:** Google Fonts, YouTube embed on homepage
- **Cookie origin:** Five analytics cookies are Adobe Analytics (`s_ac`, `s_fid`, `gpv_pn`, `s_ppv`, `s_tp`) — may be set by Adobe DTM script, not server-side
